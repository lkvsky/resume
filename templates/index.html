My Coachella logged-in desktop experience

My Coachella

My Coachella is a single page web app built on Rails, AJAX, Devise/Omniauth for authentication and the YouTube JavaScript/iframe API. Templating is all done on the client-side via Handlebars. I started with scripts that scraped Coachella's website to get band info for the artists, then fetched YouTube tracks for each artist based on their YouTube channels. Users can log-in by signing up for an account or authenticating through Facebook. If a user signs up during his/her guest session, all playlists and favorite/hated songs get transferred over from the to their newly created account. The app allows users to generate random playlists that the user can edit as he or she pleases. Users can "Like" their favorite artists to create an auto-playlist of their favorite songs, or they can "Hate" songs that they never want to show up again in their playlists. Tour the code on GitHub.

Instavention game play experience

Instavention

This is my first live Backbone app and first experience using Rails as a Backbone back-end. It's a photo guessing game using the Instagram API. Users can log in and authenticate with their Instagram credentials and play the game using their own feeds, or log in as a guest for a quick play experience with the most popular photos on Instagram. The project is currently optimized for a desktop/tablet experience, but I plan on building it out to be a more mobile-friendly experience. Keep track of my progress on my GitHub repo.

Best Coast Eats Local search for San Francisco

Best Coast Eats Local

I wrote my first project last fall: a responsive/mobile-friendly West Coast farmer's market finder and API mashup. The app was build foundationally using the Google maps API and a USDA data-driven farmer's market API. The market API is not terribly performant, so I make use of the HTML5 Local Storage API for faster repeat visits. The info windows are templatized using Handlebars and trigger geo-location-based Flickr API calls.
